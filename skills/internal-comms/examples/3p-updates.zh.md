# 3P更新说明

您被要求编写3P更新。3P更新代表"进度（Progress）、计划（Plans）、问题（Problems）"。主要受众是高管、领导、其他队友等。它们旨在非常简洁和切中要点：想想可以在30-60秒或更短时间内阅读的内容。它们也针对对团队工作有一定但不多背景的人。

3P可以涵盖任何规模的团队，从一直到整个公司。团队越大，任务应该越不细致。例如，"移动团队"可能有"发布功能"或"修复错误"，而公司可能有真正实质的3P，如"雇佣了20名新员工"或"完成了10笔新交易"。

它们代表团队在时间范围内的工作，几乎总是一周。它们包括三个部分：
1) 进度：团队在下一时间段内完成的工作。主要关注发布的内容、达到的里程碑、创建的任务等。
2) 计划：团队计划在下一时间段内做什么。关注团队最关注、真正高优先级的事情等。
3) 问题：任何减慢团队速度的事情。这可能是人太少、阻止团队前进的bug或障碍、一些失败交易等。

在编写它们之前，确保您知道团队名称。如果没有指定，您可以明确询问您为之编写的团队名称是什么。

## 可用工具
尽可能尝试从可用来源获取您需要的信息：
- Slack：团队成员的更新帖子 - 理想情况下在有很多反应的大频道中寻找帖子
- Google Drive：来自有很多浏览量的关键团队成员的文档
- 电子邮件：有很多回复或似乎相关的很多内容的电子邮件
- 日历：有很多重要性的非定期会议，如产品审查等。

尝试收集尽可能多的上下文，专注于涵盖您为之编写的时间段的事情：
- 进度：一周前到今天之间的任何事情
- 计划：今天到下一周之间的任何事情
- 问题：一周前到今天之间的任何事情

如果您没有访问权限，您可以询问用户想要涵盖的事情。他们也可能直接向您包含这些事情，在这种情况下，您主要只是为这种特定格式进行格式化。

## 工作流程

1. **明确范围**：确认团队名称和时间段（通常进度/问题为上一周，计划为下一周）
2. **收集信息**：使用可用工具或直接询问用户
3. **起草更新**：遵循严格的格式指南
4. **审查**：确保简洁（30-60秒阅读）且数据驱动

## 格式

格式总是相同的，非常严格的格式。除了这种格式外，绝不要使用任何其他格式。选择一个有趣并捕捉团队和更新氛围的emoji。

[选择emoji] [团队名称]（涵盖的日期，通常是一周）
进度：[1-3句内容]
计划：[1-3句内容]
问题：[1-3句内容]

每个部分不应超过1-3句：清晰、切中要点。它应该是数据驱动的，通常在可能的情况下包括指标。语气应该非常实际，不超级散文。
