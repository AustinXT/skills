算法哲学是通过代码表达的计算美学运动。输出.md文件（哲学）、.html文件（交互式查看器）和.js文件（生成算法）。

这分两个步骤进行：
1. 算法哲学创建（.md文件）
2. 通过创建p5.js生成艺术表达（.html + .js文件）

首先，执行此任务：

## 算法哲学创建

首先，创建一个算法哲学（不是静态图像或模板），它将通过以下方式被解释：
- 计算过程、涌现行为、数学美感
- 种子随机性、噪声场、有机系统
- 粒子、流、场、力
- 参数变化和受控混沌

### 关键理解
- 收到什么：用户提供的细微输入或指示，应予以考虑但用作基础；它不应限制创造性自由
- 创造什么：算法哲学/生成美学运动
- 接下来发生什么：同一版本接收哲学并用代码表达它——创建90%算法生成、10%基本参数的p5.js草图

考虑这种方法：
- 为生成艺术运动写一个宣言
- 下一阶段涉及编写使它活跃的算法

哲学必须强调：算法表达。涌现行为。计算美感。种子变化。

### 如何生成算法哲学

**命名运动**（1-2个词）："有机湍流" / "量子和声" / "涌现静止"

**阐述哲学**（4-6段 - 简洁但完整）：

为了捕捉算法本质，表达这种哲学如何通过以下方式体现：
- 计算过程和数学关系？
- 噪声函数和随机模式？
- 粒子行为和场动力学？
- 时间演化和系统状态？
- 参数变化和涌现复杂性？

**关键指南：**
- **避免冗余**：每个算法方面只应提及一次。避免重复噪声理论、粒子动力学或数学概念，除非添加新的深度。
- **反复强调工艺**：哲学必须多次强调最终算法应该看起来花费了无数小时开发、精心完善、来自绝对顶尖的领域专家。这种框架至关重要 - 重复短语如"精心制作的算法"、"深度计算专业知识的产物"、"费力的优化"、"大师级实现"。
- **留出创作空间**：对算法方向要具体，但足够简洁，以便下一个Claude有空间在极高工艺水平上做出解释性实现选择。

哲学必须指导下一版本以算法方式表达想法，而不是通过静态图像。美感存在于过程中，而不是最终帧中。

### 哲学示例

**"有机湍流"**
哲学：受自然法则约束的混沌，从无序中涌现的有序。
算法表达：由分层Perlin噪声驱动的流场。数千个粒子跟随向量力，它们的轨迹累积成有机密度地图。多个噪声八度创建湍流区域和安静区域。颜色从速度和密度中出现 - 快速粒子燃烧明亮，慢速粒子褪色到阴影。算法运行直到平衡 - 经过大师计算美学无数迭代精心调优的平衡，其中每个参数都被精细调整。

**"量子和声"**
哲学：离散实体表现出波状干涉模式。
算法表达：初始化在网格上的粒子，每个携带通过正弦波演化的相位值。当粒子靠近时，它们的相位干涉 - 建设性干涉创建明亮节点，破坏性干涉创建空洞。简单谐波运动产生复杂的涌现曼陀罗。费力频率校准的结果，每个比率都经过仔细选择以产生共鸣美感。

**"递归低语"**
哲学：跨尺度自相似性，有限空间中的无限深度。
算法表达：递归细分的分支结构。每个分支稍微随机化但受黄金比例约束。L系统或递归细分生成感觉既有数学性又有机的树状形式。微妙的噪声扰动打破完美对称。线条权重随着每个递归级别减少。每个分支角度都是深度数学探索的产物。

**"场动力学"**
哲学：不可见力量通过它们对物质的影响变得可见。
算法表达：由数学函数或噪声构建的向量场。粒子在边缘诞生，沿场线流动，在达到平衡或边界时死亡。多个场可以吸引、排斥或旋转粒子。可视化只显示轨迹 - 不可见力量的幽灵般证据。通过力平衡精心编排的计算舞蹈。

**"随机结晶"**
哲学：随机过程结晶为有序结构。
算法表达：随机圆打包或Voronoi镶嵌。从随机点开始，让它们通过松弛算法演化。细胞推开直到平衡。基于细胞大小、邻居数量或距离中心的颜色。出现的有机平铺感觉既随机又必然。每个种子产生独特的结晶美感 - 大师级生成算法的标志。

*这些是压缩示例。实际的算法哲学应该是4-6个实质性段落。*

### 核心原则
- **算法哲学**：创建通过代码表达的计算世界观
- **过程重于产品**：始终强调美感从算法执行中涌现 - 每次运行都是独特的
- **参数表达**：想法通过数学关系、力、行为交流 - 不是静态构图
- **艺术自由**：下一个Claude以算法方式解释哲学 - 提供创造性实现空间
- **纯生成艺术**：这是关于制作活算法，而不是带有随机性的静态图像
- **专家工艺**：反复强调最终算法必须感觉精心制作，通过无数迭代完善，是计算美学领域绝对顶尖人员的深度专业知识产物

**算法哲学应该4-6段长。**用将预期愿景汇集在一起的诗意计算哲学填充它。避免重复相同的观点。将此算法哲学作为.md文件输出。

## 推导概念种子

**关键步骤**：在实现算法之前，从原始请求中识别微妙的概念线索。

**核心原则**：
概念是**嵌入在算法本身内的微妙、专业的参考** - 不总是字面意义，总是很专业。熟悉主题的人应该直觉地感受到它，而其他人只是体验到大师级的生成组合。算法哲学提供了计算语言。推导出的概念提供了灵魂 - 编织在参数、行为和涌现模式中的静默概念DNA。

这**非常重要**：参考必须如此精致，以增强作品的深度而不宣告自己。像爵士音乐家通过算法和声引用另一首歌一样 - 只有知道的人会抓住它，但每个人都会欣赏生成美感。

建立哲学和概念框架后，通过代码表达它。在继续之前暂停收集想法。只使用创建的算法哲学和以下指令。

### ⚠️ 步骤0：先阅读模板 ⚠️

**关键：在写任何HTML之前：**

1. **阅读**`templates/viewer.html`使用Read工具
2. **研究**确切的结构、样式和Anthropic品牌
3. **使用该文件作为字面起点** - 不仅仅是灵感

**遵循这些实践：**
- ✅ 复制模板的确切HTML结构

模板是基础。建立在其上，而不是重建它。

为了创建生活呼吸的画廊质量计算艺术，使用算法哲学作为基础。

### 技术要求

**种子随机性（艺术块模式）**：
```javascript
// 始终使用种子以确保可重现性
let seed = 12345; // 或来自用户输入的哈希
randomSeed(seed);
noiseSeed(seed);
```

**参数结构 - 遵循哲学**：

为了建立从算法哲学自然涌现的参数，考虑："这个系统的哪些品质可以调整？"

```javascript
let params = {
  seed: 12345,  // 始终包含种子以确保可重现性
  // colors
  // 添加控制您的算法的参数：
  // - 数量（多少？）
  // - 尺度（多大？多快？）
  // - 概率（可能性？）
  // - 比例（什么比例？）
  // - 角度（什么方向？）
  // - 阈值（行为何时改变？）
};
```

**设计有效参数，专注于系统需要可调谐的属性，而不是从"模式类型"的角度思考。**

**核心算法 - 表达哲学**：

**关键**：算法哲学应该决定构建什么。

为了通过代码表达哲学，避免思考"我应该使用哪种模式？"而是想"如何通过代码表达这种哲学？"

如果哲学是关于**有机涌现**，考虑使用：
- 随时间累积或增长的元素
- 受自然规则约束的随机过程
- 反馈循环和交互

如果哲学是关于**数学美感**，考虑使用：
- 几何关系和比例
- 三角函数和谐波
- 创造意外模式的精确计算

如果哲学是关于**受控混沌**，考虑使用：
- 严格边界内的随机变化
- 分叉和相变
- 从无序中涌现的有序

**算法从哲学中流出，而不是从选项菜单中流出。**

为指导实现，让概念本质告知创造性和原创选择。为这个特定请求构建表达愿景的东西。

**画布设置**：标准p5.js结构：
```javascript
function setup() {
  createCanvas(1200, 1200);
  // 初始化您的系统
}

function draw() {
  // 您的生成算法
  // 可以是静态的（noLoop）或动画的
}
```

### 工艺要求

**关键**：要达到大师级，创建感觉像通过大师生成艺术家无数迭代涌现的算法。仔细调整每个参数。确保每个模式都有目的地涌现。这不是随机噪声 - 这是通过深度专业知识完善的受控混沌。

- **平衡**：复杂性而不视觉噪声，有序而不僵化
- **色彩和谐**：深思熟虑的调色板，不是随机RGB值
- **构图**：即使在随机性中，保持视觉层次和流动
- **性能**：平滑执行，如果动画则优化为实时
- **可重现性**：相同种子总是产生相同输出

### 输出格式

输出：
1. **算法哲学** - 作为markdown或文本解释生成美学
2. **单一HTML Artifact** - 从`templates/viewer.html`构建的自包含交互式生成艺术（见步骤0和下一节）

HTML artifact包含所有内容：p5.js（来自CDN）、算法、参数控件和UI - 全部在一个文件中，可立即在claude.ai artifacts或任何浏览器中工作。从模板文件开始，而不是从零开始。

## 交互式Artifact创建

**提醒**：`templates/viewer.html`应该已经被读取（见步骤0）。使用该文件作为起点。

为了允许探索生成艺术，创建一个单一的自包含HTML artifact。确保此artifact在claude.ai或任何浏览器中立即工作 - 无需设置。内联嵌入所有内容。

### 关键：固定与可变

`templates/viewer.html`文件是基础。它包含所需的确切结构和样式。

**固定（始终完全按所示包含）：**
- 布局结构（头部、侧边栏、主要画布区域）
- Anthropic品牌（UI颜色、字体、渐变）
- 侧边栏中的种子部分：
  - 种子显示
  - 上一个/下一个按钮
  - 随机按钮
  - 跳转到种子输入 + 执行按钮
- 侧边栏中的操作部分：
  - 重新生成按钮
  - 重置按钮

**可变（为每个艺术品自定义）：**
- 整个p5.js算法（setup/draw/类）
- 参数对象（定义艺术需要什么）
- 侧边栏中的参数部分：
  - 参数控件数量
  - 参数名称
  - 滑块的最小/最大/步长值
  - 控件类型（滑块、输入等）
- 颜色部分（可选）：
  - 有些艺术需要颜色选择器
  - 有些艺术可能使用固定颜色
  - 有些艺术可能是单色的（不需要颜色控件）
  - 根据艺术需求决定

**每个艺术应该有独特的参数和算法！**固定部分提供一致的UX - 其他一切都表达独特愿景。

### 必需功能

**1. 参数控件**
- 数字参数的滑块（粒子数量、噪声尺度、速度等）
- 调色板颜色的颜色选择器
- 参数变化时的实时更新
- 恢复默认值的重置按钮

**2. 种子导航**
- 显示当前种子编号
- "上一个"和"下一个"按钮循环遍历种子
- "随机"按钮随机种子
- 跳转到特定种子的输入字段
- 请求时生成100种变化（种子1-100）

**3. 单一Artifact结构**
```html
<!DOCTYPE html>
<html>
<head>
  <!-- 来自CDN的p5.js - 始终可用 -->
  <script src="https://cdnjs.cloudflare.com/ajax/libs/p5.js/1.7.0/p5.min.js"></script>
  <style>
    /* 所有样式内联 - 干净、最小 */
    /* 画布在顶部，控件在下方 */
  </style>
</head>
<body>
  <div id="canvas-container"></div>
  <div id="controls">
    <!-- 所有参数控件 -->
  </div>
  <script>
    // 所有p5.js代码在这里内联
    // 参数对象、类、函数
    // setup() 和 draw()
    // UI处理器
    // 一切自包含
  </script>
</body>
</html>
```

**关键**：这是单一artifact。没有外部文件，没有导入（除了p5.js CDN）。一切内联。

**4. 实现细节 - 构建侧边栏**

侧边栏结构：

**1. 种子（固定）** - 始终完全按所示包含：
- 种子显示
- 上一个/下一个/随机/跳转按钮

**2. 参数（可变）** - 为艺术创建控件：
```html
<div class="control-group">
    <label>参数名称</label>
    <input type="range" id="param" min="..." max="..." step="..." value="..." oninput="updateParam('param', this.value)">
    <span class="value-display" id="param-value">...</span>
</div>
```
根据参数数量添加尽可能多的control-group div。

**3. 颜色（可选/可变）** - 如果艺术需要可调颜色则包含：
- 如果用户应该控制调色板，添加颜色选择器
- 如果艺术使用固定颜色，跳过此部分
- 如果艺术是单色的，跳过

**4. 操作（固定）** - 始终完全按所示包含：
- 重新生成按钮
- 重置按钮
- 下载PNG按钮

**要求**：
- 种子控件必须工作（上一个/下一个/随机/跳转/显示）
- 所有参数必须有UI控件
- 重新生成、重置、下载按钮必须工作
- 保持Anthropic品牌（UI样式，不是艺术颜色）

### 使用Artifact

HTML artifact立即工作：
1. **在claude.ai中**：显示为交互式artifact - 立即运行
2. **作为文件**：保存并在任何浏览器中打开 - 不需要服务器
3. **分享**：发送HTML文件 - 它完全自包含

## 变化与探索

artifact默认包括种子导航（上一个/下一个/随机按钮），允许用户无需创建多个文件即可探索变化。如果用户想要突出显示特定变化：

- 包括种子预设（"变化1：种子42"、"变化2：种子127"等按钮）
- 添加"画廊模式"，并排显示多个种子的缩略图
- 全部在同一个单一artifact内

这就像从同一个版创建一系列印刷品 - 算法是一致的，但每个种子都揭示了其潜力的不同方面。交互性质意味着用户通过探索种子空间发现自己的最爱。

## 创作过程

**用户请求** → **算法哲学** → **实现**

每个请求都是独特的。过程涉及：

1. **解释用户意图** - 正在寻求什么美学？
2. **创建算法哲学**（4-6段）描述计算方法
3. **在代码中实现它** - 构建表达这种哲学的算法
4. **设计适当参数** - 什么应该是可调的？
5. **构建匹配的UI控件** - 那些参数的滑块/输入

**常量**：
- Anthropic品牌（颜色、字体、布局）
- 种子导航（始终存在）
- 自包含HTML artifact

**其他一切都是可变的**：
- 算法本身
- 参数
- UI控件
- 视觉结果

为了达到最佳结果，相信创造力并让哲学指导实现。
