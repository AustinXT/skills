# 代理技能规范

技能是指令、脚本和资源的文件夹，代理可以动态发现和加载这些内容以在特定任务上表现更好。为了使文件夹被识别为技能，它必须包含一个`SKILL.md`文件。

# 技能文件夹布局

一个最小的技能文件夹如下所示：

```
my-skill/
  - SKILL.md
```

更复杂的技能可以根据需要添加额外的目录和文件。

# SKILL.md文件

技能的"入口点"是`SKILL.md`文件。它是唯一必须存在的文件。该文件必须以YAML frontmatter开头，然后是常规markdown。

## YAML Frontmatter

YAML frontmatter有2个必需属性：

- `name`
    - 技能的名称，使用连字符格式
    - 受限于小写Unicode字母数字 + 连字符
    - 必须与包含SKILL.md的目录名称匹配
- `description`
    - 技能功能和Claude何时应该使用它的描述

有3个可选属性：

- `license`
    - 应用于技能的许可证
    - 我们建议保持简短（许可证名称或捆绑许可证文件的名称）
- `allowed-tools`
    - 预先批准运行的工具列表
    - 目前仅在Claude Code中支持
- `metadata`
    - 从字符串键到字符串值的映射
    - 客户端可以使用它来存储代理技能规范未定义的额外属性
    - 我们建议使您的键名合理唯一以避免意外冲突

## Markdown主体

Markdown主体没有限制。
